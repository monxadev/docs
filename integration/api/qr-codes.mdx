## Introduction

Embedded QR Code Payments let you accept QR-based transactions **directly inside your platform** without redirecting customers to external pages.

At a broad level, incorporating QR code payments into your system follows a four-step process:

<Columns cols={2}>
  <Card title="Create a Payment Intent" icon="star" iconType="solid" href="#">
    Start by creating a payment intent in your system with the required details.
  </Card>
  <Card title="Generate & Display the QR Code" icon="qrcode" iconType="solid" href="#">
    Use the payment intent to generate a QR code and present it to the customer.
  </Card>
  <Card title="Customer Scans to Pay" icon="camera" iconType="solid" href="#">
    The customer scans the QR code and completes payment through their banking or wallet app.
  </Card>
  <Card title="Handle Webhooks" icon="anchor" iconType="solid" href="#">
    After processing, Monxa sends a webhook notification. Use this to update your server and client with the final payment status.
  </Card>
</Columns>

## Integration Flow

### Step 1: Create a Payment Intent

This is the initial stage of the payment process. After gathering all the userâ€™s details and preparing to collect payment, use this API to generate a payment request.

<Tip>
  **Endpoint: POST /payment-intents**
</Tip>

<AccordionGroup>
  <Accordion title="Request Parameters" iconType="solid">
    | Parameter             | Type    | Description                                                  |
    | --------------------- | ------- | ------------------------------------------------------------ |
    | **amount**            | string  | _Required._ The amount to be paid.                           |
    | **currency**          | string  | _Required._ The currency of the payment (e.g., â€œsgdâ€).       |
    | **payment_methods[]** | array   | _Required._ Specify **paynow_online** as the payment method. |
    | **generate_qr**       | boolean | _Required._ Set to **true** to generate the QR code.         |
    | **name**              | string  | Optional. The name of the customer.                          |
    | **email**             | string  | Optional. The email of the customer.                         |
    | **phone**             | string  | Optional. The phone number of the customer.                  |
    | **purpose**           | string  | Optional. The purpose of the payment.                        |
    | **reference_number**  | string  | Optional. A reference number for the payment.                |
  </Accordion>
  <Accordion title="Example Request">
    ```json
    {
      "amount": "123.00",
      "currency": "sgd",
      "payment_methods": ["qr_qris"],
      "generate_qr": true,
      "expiry_date": "2024-06-01T14:37:11"
    }
    ```
  </Accordion>
</AccordionGroup>

### Step 2: Generate and Present the QR Code

After creating the payment request, use the response data to generate a QR code and present it to the user.

<Warning>
  **ğŸ” Sandbox Behavior**

  - In the **Sandbox environment**, the `qr_code` value returned may be a URL (e.g., `https://checkout.monxa.dev/...`) instead of the raw PayNow QR string.
  - This allows you to test the flow easily using any standard QR code scanner or by opening the link directly in a browser.

  **ğŸš€ Production Behavior**

  - In **Production**, the `qr_code` value will be a **raw PayNow payload string**.
  - You must convert this string into a scannable QR code for use with supported banking apps.
</Warning>

### Step 3: Client Scans and Pay

The customer scans the QR code with their banking app to initiate and confirm the payment.

### Step 4: Handle Webhooks & Update Payment Status

After the payment is completed, your system must handle **webhooks** to receive payment notifications and keep the order status in sync.
